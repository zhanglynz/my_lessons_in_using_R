2.1 Counting is not that easy

Finding number of rows under some constraints and then getting some a ratio is a common practice for an analyst who is doing official statistics. But sometimes counting is not that easy when a record is not the same as a response. An example is that people are allowed to make multiple choices. We use the following fake data to tell two stories.

library(readr)
the_url <- "https://raw.githubusercontent.com/LarryZhang2016/Data/master/fake_data_4_counting_example.csv"
fake_data <- read_csv(the_url)

Story 1: Find the number of respondents. The code below is wrong!

library(dplyr)
no_of_respondents <-
  fake_data %>% 
  summarise(n = n())

print.data.frame(no_of_respondents)
##    n
## 1 39

Here we should use n_distinct(), and the correct code is:

library(dplyr)
no_of_respondents <-
  fake_data %>% 
  summarise(n = n_distinct(ID))

print.data.frame(no_of_respondents)
##   n
## 1 5

Story 2: Tally the data with one dimension be Question-Number and the other dimension be Answers, and then create another table, which shows the proportions of people made the choice.

no_of_people_choose_A_4_Q1 <-
  fake_data %>% 
  group_by(Q_NO, answers) %>% 
  tally() %>% 
  tidyr::spread(key = answers, value = n)

print.data.frame(no_of_people_choose_A_4_Q1)
##   Q_NO A B C
## 1    1 4 2 3
## 2    2 5 3 2
## 3    3 3 3 2
## 4    4 5 3 4
proportion_table <- 
  mutate(no_of_people_choose_A_4_Q1, 
         A = A / no_of_respondents$n,
         B = B / no_of_respondents$n,
         C = C / no_of_respondents$n) 

print.data.frame(proportion_table)
##   Q_NO   A   B   C
## 1    1 0.8 0.4 0.6
## 2    2 1.0 0.6 0.4
## 3    3 0.6 0.6 0.4
## 4    4 1.0 0.6 0.8

Note that the denominator should be 5 instead of 39.

2.2 Joining tables, be careful!

When the values for one key variable are not unique, we must pay more attention to what we get from joining two tables – sometimes it’s not really what we want.

Example 1:

library(dplyr)

(a_table <- data_frame(x = letters[1:5], y = 1:5))
## # A tibble: 5 x 2
##   x         y
##   <chr> <int>
## 1 a         1
## 2 b         2
## 3 c         3
## 4 d         4
## 5 e         5
(b_table <- data_frame(x = c("a", "b", "b", "a", "c")))
## # A tibble: 5 x 1
##   x    
##   <chr>
## 1 a    
## 2 b    
## 3 b    
## 4 a    
## 5 c

Here we want to use x as the key and find “overlap” between a_table and b_table. We may write down

(c_table <- inner_join(a_table, b_table, by = "x"))
## # A tibble: 5 x 2
##   x         y
##   <chr> <int>
## 1 a         1
## 2 a         1
## 3 b         2
## 4 b         2
## 5 c         3

but this code does not give the result that we want. so the correct code should be

temp_b_table <-
  b_table %>% 
  distinct(x)
  
c_table <-
  a_table %>% 
  inner_join(temp_b_table, by = "x")

(c_table)
## # A tibble: 3 x 2
##   x         y
##   <chr> <int>
## 1 a         1
## 2 b         2
## 3 c         3

Or, we can use semi_join().

c_table <-
  a_table %>% 
  semi_join(b_table, by = "x")

(c_table)
## # A tibble: 3 x 2
##   x         y
##   <chr> <int>
## 1 a         1
## 2 b         2
## 3 c         3

Example 2:

a_table <- data_frame(x = letters[1:5], y = 1:5)
d_table <- data_frame(x = c("a", "b", "b", "a", "c"), 
                      z = 6:10)

e_table <-
  a_table %>% 
  left_join(d_table, by = "x") 
(e_table)
## # A tibble: 7 x 3
##   x         y     z
##   <chr> <int> <int>
## 1 a         1     6
## 2 a         1     9
## 3 b         2     7
## 4 b         2     8
## 5 c         3    10
## 6 d         4    NA
## 7 e         5    NA

Note that values of x in e_table are not unique and e_table has more rows than a_table.

2.3 Using group_by and arrange

“grouped arrange() is the same as ungrouped; unless you set .by_group = TRUE, in which case it orders first by the grouping variables” (Introduction to dplyr)

The following example shows the difference between using .by_group = TRUE and withou using it.

library(dplyr)

my_func <- function(x)
{n <- sum(is.na(x))
 if(n == 0) return(x)
 L <- length(x)
 x[(L+1-n):L] <- x[L-n] + (1:n)
 return(x)
}

temp_v <- c('a', 'b', letters[1:3], 'b', 'c', letters[1:3])
L <- length(temp_v)
df <- data.frame(x = temp_v,
                 y = c((L-3):1, NA, NA, NA))

(df)
##    x  y
## 1  a  7
## 2  b  6
## 3  a  5
## 4  b  4
## 5  c  3
## 6  b  2
## 7  c  1
## 8  a NA
## 9  b NA
## 10 c NA
# frist attempt, without using .by_group = TRUE
# surprisingly (goo!) we can get what we want
df_new_1 <- 
  df %>% 
  group_by(x) %>% 
  arrange(y,) %>% 
  mutate(z = my_func(y))

(df_new_1)
## # A tibble: 10 x 3
## # Groups:   x [3]
##    x         y     z
##    <chr> <int> <int>
##  1 c         1     1
##  2 b         2     2
##  3 c         3     3
##  4 b         4     4
##  5 a         5     5
##  6 b         6     6
##  7 a         7     7
##  8 a        NA     8
##  9 b        NA     7
## 10 c        NA     4
# second try, using .by_group = TRUE
# this is the code supposed to be
df_new_2 <- 
  df %>% 
  group_by(x) %>% 
  arrange(y, .by_group = TRUE) %>% 
  mutate(z = my_func(y))

(df_new_2)
## # A tibble: 10 x 3
## # Groups:   x [3]
##    x         y     z
##    <chr> <int> <int>
##  1 a         5     5
##  2 a         7     7
##  3 a        NA     8
##  4 b         2     2
##  5 b         4     4
##  6 b         6     6
##  7 b        NA     7
##  8 c         1     1
##  9 c         3     3
## 10 c        NA     4
identical(arrange(df_new_1, x), df_new_2)
## [1] TRUE
